<h2 id="data-viva"> Data Viva

#### [Retornar para o índice](#indice)

### Descrição
 - Dados socioeconômicos de mais de 5000 municípios brasileiros;
 - Bases de dados do Munistério do Trabalho e Previdência Social, ministério do desenvolvimento, indústria e comércio exterior e ministério da educação;
 - Dados nacionais dos últimos dez anos;
 - Ao fornecer acesso aos dados oferece um conhecimento mais detalhado da economia brasileira;
 - Existem pesquisas por perfis e áreas
   - Perfis
     1. Empreendedores
     2. Agentes de Desenvolvimento
     3. Estudantes e profissionais
   - Áreas
     1. Localidades Brasileiras
     2. Ocupações
     3. Atividades Econômicas
     4. Produtos
     5. Parceiros Comerciais
     6. Universidades
     7. Ensino Superior
     8. Cursos Básicos

### Link
http://www.dataviva.info/pt/


### Slogan
"A maior plataforma de visualização de dados sociais e econômicos do Brasil. Totalmente aberta e livre."

### Fluxo da aplicação
**Ao pesquisar por área**
  1. Na navbar, selecione a opção "PESQUISE"
  2. Acesse o link "INICIE UMA PESQUISA" na aba recursos básicos
  3. Selecione uma área
  4. Selecione uma pergunta
  5. Defina um escopo para pergunta
  6. Visualize os indicadores

**Ao pesquisar por base de dados**
  1. Na navbar, selecione a opção "PESQUISE"
  2. Acesse o link "CONSTRUA GRÀFICOS" na aba recursos avançados
  3. Selecione a base de dados
  4. Filtre pelas informações desejadas, existem diferentes opções de filtros para as quatro base de dados disponíveis.

### Desvantagens do DataViva
  - Gráficos de dificíl interpretação na pesquisa por perfil e por área
  - Não pode inserir base de dados
  - Somente dados socioeconômicos dos municípios brasilieros
  - Não é colaborativa
  - O cadastro não faz sentido, me cadastrei mas nenhuma opção nova foi habilitada, continuo com acesso as mesmas funcionalidades disponíveis antes do cadastro.
  - É uma ferramenta somente de visualização de dados já consolidados, não possibilita trabalhar com outros tipos de dados.
  - Não é colaborativa

<h2 id="google-drive"> Google Drive

#### [Retornar para o índice](#indice)

### Descrição
Google Drive é um serviço de armazenamento e sincronização de arquivos que foi apresentado pela Google em 24 de abril de 2012.

"Seu armazenamento do Google Drive
Sua Conta do Google vem com armazenamento gratuito do Google Drive para compartilhamento no Gmail, Google Fotos, Documentos Google e muito mais. Se isso não for suficiente, você poderá adicionar mais armazenamento aqui." (Obtido a partir de configurações de armazenamento)

### Funcionalidades:
  - Gerar documentos google
  - Opções de favoritar
  - Verificar acessos recentes
  - Sincronizar com o computador
  - Backup de informações
  - Compartilhamento com permissão de edição
  - Compartilhamento com permissão de visualização

### Funcionalidades - Google Sheets
  - CRUD tabelas, CRUD de células e CRUD de colunas
  - Gera alguns tipos de gráficos
  - Formatação de fontes
  - Formas de desenho
  - Adição de comentários
  - Versionamento de histórico
  - Importa arquivo para editar
  - Salva arquivo nos formatos: .xls, .ods, .pdf, .html(zipado), .csv, .tsv
  - Compartilha com permissão de visualização
  - Compartilha com permissão de edição

### Aspectos favoráveis
- Web app, app mobile
- Ferramenta gratuita, com opção premium
- Compartilhamento no Gmail, Google Fotos, Documentos Google
- Edição colaborativa
- Registro de histórico
- Diferentes formatos de download
- Acesso às informações em diferentes dispositivos, uma vez que está em nuvem.

### Aspectos frágeis
- O acesso de compartilhamento é dado por link ou confirmação por email, que para o contexto do NMT pode não ser muito interessante uma vez que o objetivo é facilitar o acesso ao público. Um web site pode ser mais interessante para essa finalidade.
- Com permissão apenas de visualização no Google Sheets não é possível gerar o gráfico no mesmo arquivo. É possível ter acesso ao que está disponibilizado, porém para criar um gráfico, ou o usuário faz o download e gera em outro contexto ou copia as informações e utiliza em outro contexto. O usuário pode conseguir realizar essa atividade de forma mais
simplificada com o Observ, com geração direta de gráfico.

<h2 id="sciehub">SCIEHUB

#### [Retornar para o índice](#indice)

### Link
https://scihub.org/

### Slogan
"Nós somos o guia no mundo da ciência liderando a rodovia da informação"

### Definição
O Sci-Hub é um repositório online com mais de 64 milhões artigos científicos, disponíveis no seu website. Novos documentos são enviados diariamente através de domínios de instituições de ensino,que contornam sistemas que restringem o acesso a usuários da Internet sem registros pagos em seus sites. Ele foi fundado pela neurocientista cazaque Alexandra Elbakyan com o objetivo de melhorar a popularização do conhecimento, possibilitando o acesso de mais pessoas a conteúdo pago.
Fornecer acesso à pesquisa para instituições com menos privilégios é o objetivo expresso de Elbakyan's na criação desse site.
Voltada para uma escala mundial.

### Funcionalidades
- Não requer pagamento
- Não requer cadastro
- Possui vários conteúdos de jornais, revistas, períodicos.
- Publicação de periódicos

### Aspectos frágeis
- Na tela de pesquisa existem vários anúncios e no final, por último, o campo de pesquisa é exibido.
- Página lenta para carregar
- Não possui estrutura voltada para geração de gráficos
- Não é colaborativa

<h2 id="info-dengue"> Info Dengue

#### [Retornar para o índice](#indice)

### Link
https://info.dengue.mat.br/alerta/ES/dengue

### Descrição
O Sistema InfoDengue visa gerar um sistema de alerta em tempo quase-real para os casos de transmissão de dengue nos municípios. A ideia principal é o acesso rápido a informação!
Para que o sistema funcione, diversas metodologias de análise são utilizadas de forma integrada, incluindo modelagem de sobrevida, modelagem bayesiana, e um pipeline de coleta e processamento rápido de dados. Essa automatização integrada com modelagem estatística possibilita a rápida visualização da distribuição dos casos de transmissão de dengue em um município ao longo do tempo e do espaço. Os dados que alimentam o sistema são provenientes de diferentes fontes, incluindo clima, redes sociais e dados oficiais de notificação.

### Aspectos favoráveis
- Riqueza de detalhes sobre a dengue
- Exibição de
- Gera o indicador nos formatos: .png, .jpeg, .pdf, .svg

### Aspectos frágeis

- Falta de feedback quando o usuário erra. Exemplo digitei um texto na busca o e nenhuma mensagem foi emitida.
- Renderização da lista é prejudicada, não aparece por completo.
- Não é colaborativo, apenas consultivo
- Campo de Estado fica em outro contexto do campo do município. Pode ser ruim pra usabilidade.
